# Template Governance Runbook

## When SACST Changes

1. Update canonical docs and schemas first.
2. Update templates and adapter generation.
3. Run template validation and integration tests.
4. Record any migration impact.
5. Only then sync the new template into project repos.

## Promotion Rule

Successful project patterns may become candidate improvements, but they are promoted into SACST only after sanitization, review, validation, and explicit approval.
